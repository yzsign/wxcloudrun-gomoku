五子棋积分系统设计文档
版本：v1.1
适用场景：在线五子棋对战平台
核心算法：基于 Elo 的改进型积分系统

1. 概述

1.1 积分类型（双层，可只实现天梯分）

类型	字段建议	作用	说明
天梯分（竞技积分）	elo_score	匹配、段位、排行榜	本文档第 2–11 节；胜负即变动，不可直接购买
活跃积分（可选）	activity_points	商城/兑换装饰/任务	与棋力无关，不参与 Elo 公式；规则见第 12 节

1.2 概述（天梯分）
本积分系统用于评估玩家棋力水平，并保证公平匹配。天梯分变动综合考虑：

对手实力（Elo 预期胜率）

对局完整度（步数）

连续胜负状态（连胜/连败修正）

特殊对局结果（逃跑、平局、越级挑战）

2. 初始积分与定级
项目	值
新用户初始积分	1200
定级对局数	10 盘有效对局（无逃跑/超时），用于判定是否仍处「定级期」
定级后积分调整幅度	恢复正常（无额外保护）
定级期 / 新用户：匹配范围限制更严；保护期长度见 6.3（可与定级对局数不同，分别计数）

3. 积分变动核心公式

```
R_new = R_old + K × (S - E) × M × C
```

参数说明
符号	含义	取值范围/计算方式
R_old	对局前积分	整数
S	实际结果	胜=1，负=0，平=0.5
E	预期胜率	1 / (1 + 10^((R_opp - R_old)/400))
K	基础变动系数	根据 R_old 查表（见 3.1）
M	对局时长系数	根据总步数（见 3.2）
C	连输/连胜修正系数	根据连续胜负场数（见 3.3）
3.1 K 值表（积分区间）
积分区间	K 值
< 1400	40
1400 – 1999	30
2000 – 2399	20
≥ 2400	15
3.2 M 值（对局步数）
总步数（双方合计落子）	M 值
≥ 30	1.0
15 – 29	0.8
≤ 14	0.5
步数从第 1 手开始计数，包括认输前的最后一步。

3.3 C 值（连续胜负修正）
连胜修正（本局胜）：
连续胜场 ≥ 3 → C = 1.1 + 0.02 × (连续胜场 - 3)，上限 1.3

连败修正（本局负）：
连续负场 ≥ 3 → C = 1.1 + 0.02 × (连续负场 - 3)，上限 1.3

其他情况：C = 1.0

平局不中断连胜/连败计数（也不增加计数）。

3.4 积分取整规则
计算后四舍五入取整

最低积分：100（低于 100 则置为 100）

最高无上限

4. 特殊对局结果处理
4.1 平局（和棋）
双方 S = 0.5

积分变动正常按公式计算（E 为预期胜率，M 按实际步数）

双方 不改变 连胜/连败计数

触发条件（任一）：

双方同意和棋

棋盘满子且无五子连珠（200 手）

系统检测到重复局面循环（如长捉无意义走法）

4.2 逃跑 / 超时负
逃跑方：S = 0，积分变动正常计算，并 额外扣除 10 分

对方：S = 1，积分变动正常计算，且 强制 M = 1.0

逃跑方积分最低不低于 100

逃跑率 = 逃跑局数 / 总局数（> 15% 进入低信誉池）

4.3 断线重连
断线 1 分钟内重连 → 对局继续，无惩罚

超过 1 分钟未重连 → 判负（同超时负）

6. 匹配机制中的积分应用
6.1 常规匹配
优先匹配积分差 ≤ 200 分的对手

等待 30 秒后放宽至 ≤ 400 分

等待 60 秒后完全放开（仍不匹配跨段位挑战）

6.2 越级挑战奖励（积分差 ≥ 400 分）
低分方获胜：额外获得 min(20, floor(积分差/20)) 分（从系统池奖励，不扣高分方）

高分方获胜：按基础公式计算（低分方正常扣分）

6.3 新用户保护（前 20 盘有效对局，与 §2「定级 10 盘」独立计数）
仅在此 20 盘内：只匹配积分差 ≤ 300 分的对手

逃跑不额外扣分（但记录次数，超过 3 次取消保护）

7. 防作弊与公平性规则
7.1 连胜/连败单日限制
单日净增/净减上限：150 分

超出部分暂存于“暂存池”，次日 0 点自动释放（一次性加到积分上）

7.2 短对局刷分检测
条件：
同一 IP 或设备对局，24 小时内 ≥ 3 局，且每局总步数 < 20 手。

处罚：

清空双方该时段所有积分变动

各额外扣除 50 分（最低不低于 100 分）

7.3 AI 代打检测
查实后：

积分重置为 1000 分

封禁匹配 7 天

7.4 低信誉池（逃跑率 > 15%）
仅与同池用户匹配

单局 K 值减半（K = K/2，向下取整）

8. 排行榜与赛季重置
8.1 实时积分
对局结束后立即更新积分、段位、连胜/连败计数

8.2 天梯榜排序规则
积分降序

同分按胜率降序

胜率相同按总局数降序

再相同按注册时间升序（早注册靠前）

8.3 周进步榜
统计本周净增积分最高的 100 名玩家

发放“势如破竹”徽章（无积分奖励）

8.4 赛季重置（每 3 个月）
仅对赛季末积分 > 2200 的玩家：

```
新赛季积分 = 2200 + (赛季末积分 - 2200) × 0.8
```

积分 ≤ 2200 的玩家保持不变。

赛季重置时可选：活跃积分（若有）不清零，或按产品策略单独处理。

连胜/连败计数清零。

9. 伪代码（供 Cursor 实现参考）

```python
def calculate_elo_change(winner_elo, loser_elo, steps, winner_streak, loser_streak, is_runaway=False):
    # 通用计算 delta
    def delta(elo, opp_elo, result, steps, streak, is_winner_side):
        # result: 1=win, 0=loss, 0.5=draw
        E = 1 / (1 + 10 ** ((opp_elo - elo) / 400))
        K = get_k_value(elo)
        M = get_m_value(steps)
        C = get_c_value(streak, result)
        raw = K * (result - E) * M * C
        return round(raw)

    # 逃跑处理
    if is_runaway:
        loser_delta = delta(loser_elo, winner_elo, 0, steps, loser_streak, False) - 10
        winner_delta = delta(winner_elo, loser_elo, 1, steps, winner_streak, True)
        # 强制 winner M=1
        # 实现时可单独重算 winner 的 M=1
    else:
        winner_delta = delta(winner_elo, loser_elo, 1, steps, winner_streak, True)
        loser_delta = delta(loser_elo, winner_elo, 0, steps, loser_streak, False)

    # 积分边界处理
    winner_new = max(100, winner_elo + winner_delta)
    loser_new = max(100, loser_elo + loser_delta)

    # 单日限制（需存储当日累计变动）
    apply_daily_cap(winner_id, winner_new - winner_elo)
    apply_daily_cap(loser_id, loser_new - loser_elo)

    return winner_new, loser_new
```

10. 计算示例
示例 1：普通对局
A: 1500 分，B: 1800 分
A 胜，步数 35 手
A 无连胜，B 无连败

```
E_A = 1/(1+10^((1800-1500)/400)) ≈ 0.15
K_A = 30, M=1, C=1
Δ_A = 30*(1-0.15)*1*1 = 25.5 → 26
A_new = 1526
B_new = 1800 - 19 = 1781
```

示例 2：短对局 + 连胜
A: 2100 分，4 连胜，对手 2000 分
A 胜，步数 12 手

```
E_A = 1/(1+10^((2000-2100)/400)) ≈ 0.64
K=20, M=0.5, C=1.1
Δ_A = 20*(1-0.64)*0.5*1.1 = 3.96 → 4
A_new = 2104
对手扣 1 分（1999）
```

示例 3：越级挑战
C: 1300 分，D: 1900 分（差 600）
C 胜，步数 40 手，C 无连胜

```
E_C = 1/(1+10^((1900-1300)/400)) ≈ 0.03
K_C=40, M=1, C=1
Δ_C_base = 40*(1-0.03)*1*1 = 38.8 → 39
额外奖励 = min(20, floor(600/20)) = 20
C_new = 1300 + 39 + 20 = 1359
D_new = 1900 - 39 = 1861
```

12. 活跃积分（可选，与天梯分独立）

若产品需要「可消耗、可运营」的点数，建议单独字段 activity_points，规则示例（可按运营调整）：

项目	建议值
初始活跃积分	0
每日登录	+5（每日一次）
完成任意一局有效对局（步数 ≥ 15 且非逃跑）	+10
胜局额外	+5
上限	单日通过「对局」途径最多 +80，登录不计入该上限

用途：兑换头像框、棋盘皮肤、入场特效等；不可用于匹配或折算为天梯分。

13. 对局结算与实现前提

天梯分必须在「能唯一识别双方用户」的对局上结算：对局记录需关联 black_user_id、white_user_id（或等价 openid）。若当前仅为匿名房间 token，需在创建房间 / 匹配成功时写入双方用户标识后再落库。

可选增加 rating_change_log：user_id、room_id、delta、elo_before、elo_after、对手 id、是否逃跑、步数、对局时间，便于审计与客服处理。
